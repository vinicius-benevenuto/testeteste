PROMPT – AJUSTES NO SISTEMA PTI AUTOMATIZADO

Contexto:
Você é um analista/desenvolvedor de sistemas especialista em automação de processos, UX e regras de negócio.  
Seu objetivo é realizar ajustes funcionais e visuais no sistema descrito abaixo, seguindo rigorosamente os requisitos.

1. Nome do Sistema
- O nome correto do sistema é: PTI AUTOMATIZADO
- Substituir qualquer menção a “VIVOHUB” por “PTI AUTOMATIZADO” em toda a interface e fluxos.

2. Ajustes de Interface – Central Atacado e Central Engenharia
- Remover completamente as seguintes opções:
  • Todos
  • Rascunhos
  • Enviados
  • Em revisão
  • Aprovados

- No topo da tela, remover:
  • Central
  • Formulários
- Manter apenas a opção:
  • Sair

3. Padronização nas Demais Abas
- Aplicar as mesmas remoções de filtros, abas e itens de menu em todas as outras abas do sistema.
- Garantir padronização visual e funcional em toda a aplicação.

4. Botão de Validação
- Atualmente existem os botões:
  • Abrir
  • Gerar Excel

- Criar um novo botão chamado:
  • Validar

- O botão “Validar” deve ficar posicionado ao lado do botão “Gerar Excel”.
- Função do botão “Validar”:
  • Aprovar o PTI
  • Garantir que PTIs aprovados apareçam corretamente para o time do Atacado
  • Hoje, PTIs aprovados não aparecem para o Atacado — esse problema deve ser corrigido.

5. Gestão de Versões
- As versões do PTI devem aparecer no arquivo Excel.
- Ao gerar o Excel:
  • Deve existir uma aba chamada “Versões”
  • Cada versão deve conter os dados completos referentes àquela versão específica
- Todas as versões devem estar no mesmo arquivo Excel.
- Caso o usuário queira visualizar versões anteriores pelo sistema:
  • Manter o acesso por operadora aos formulários antigos
  • Esse recurso serve apenas para consulta histórica, não substitui o Excel

6. Pesquisa Global de PTIs
- Deve ser possível visualizar todos os PTIs gerados por meio de uma pesquisa.
- A pesquisa deve permitir encontrar qualquer PTI existente no sistema.
- Recomenda-se que a pesquisa seja simples, clara e eficiente.

Objetivo Final:
Entregar um sistema mais limpo, organizado e funcional, com:
- Interface padronizada
- Validação clara de PTIs
- Versionamento consolidado no Excel
- Visibilidade correta dos PTIs aprovados para o Atacado
- Pesquisa eficiente de todos os PTIs gerados
