QUERO QUE VOCÊ AJA COMO UM DESENVOLVEDOR SÊNIOR ESPECIALIZADO EM PYTHON E TRATAMENTO DE DADOS EM GRANDE ESCALA.
CRIE UM PROGRAMA COMPLETO, OTIMIZADO E ALTAMENTE COMENTADO, PARA IDENTIFICAR OPERADORA POR PREFIXO E VERIFICAR PORTABILIDADE.

O PROGRAMA DEVE TER UM BLOCO INICIAL ONDE EU POSSA EDITAR OS CAMINHOS DOS ARQUIVOS:

CAMINHO_ARQUIVO_1 = "CAMINHO/DO/ARQUIVO_1.EXT"         # BILHETAGEM (CONTÉM A_BUSCA E B_BUSCA)
CAMINHOS_PREFIXOS = [
    "CAMINHO/DO/PREFIXO_FIXO.csv",
    "CAMINHO/DO/PREFIXO_MOVEL.csv",
    "CAMINHO/DO/PREFIXO_SME.csv",
    "CAMINHO/DO/PREFIXO_OUTRO.csv"
]                                                      # 4 ARQUIVOS DE PREFIXOS (CSV; ;)
CAMINHO_ARQUIVO_3 = "CAMINHO/DA/PORTABILIDADE.EXT"     # BASE DE PORTABILIDADE

OBS IMPORTANTE:
OS ARQUIVOS SÃO EXTREMAMENTE GRANDES (MILHÕES DE LINHAS). O SCRIPT DEVE SER OTIMIZADO PARA BAIXO USO DE MEMÓRIA E ALTO DESEMPENHO.

============================================================
DESCRIÇÃO E REGRAS DE CADA FONTE
============================================================

ARQUIVO 1 – BILHETAGEM
- CONTÉM AS COLUNAS: A_BUSCA E B_BUSCA (EXTRAIR TODOS OS VALORES).
- EXEMPLO DE CABEÇALHO (APENAS REFERÊNCIA):
  [AS_ITX,DATA_HORA_INICIO,...,A_BUSCA,B_BUSCA,...]
- O SCRIPT DEVE CONSEGUIR LER .CSV (DELIMITADOR=;) OU .XLSX.
- TRATAR TIPOS COMO STRING PARA NÃO PERDER ZEROS À ESQUERDA.

ETAPA 2 – BASES DE PREFIXOS (4 ARQUIVOS CSV COM ‘;’)
- CADA ARQUIVO TEM CABEÇALHO NO FORMATO:
  "# Nome da Prestadora;CNPJ da Prestadora;Código Nacional;Prefixo;Faixa Inicial;Faixa Final;Status"
- EXEMPLO DE LINHAS:
  "Claro NXT Telecomunicações LTDA;66970229000167;11;7801;0000;0999;1"
  "Claro NXT Telecomunicações LTDA;66970229000167;11;7801;1000;1999;1"
  "Claro NXT Telecomunicações LTDA;66970229000167;11;7801;2000;2999;1"
- REGRAS DE LEITURA E NORMALIZAÇÃO:
  1) DELIMITADOR ‘;’, ENCODING UTF-8 COM FALLBACK PARA LATIN-1 SE NECESSÁRIO.
  2) REMOVER O CARACTERE ‘#’ DO INÍCIO DO NOME DE COLUNA, SE EXISTIR.
  3) PADRONIZAR NOMES DE COLUNAS (SEM ACENTOS, TRIM, LOWERCASE) PARA:
     nome_prestadora | cnpj_prestadora | codigo_nacional | prefixo | faixa_inicial | faixa_final | status
  4) MANTER codigo_nacional E prefixo COMO STRING (PRESERVAR ZEROS).
  5) CONVERTER faixa_inicial/faixa_final PARA INTEIRO.
  6) CONSIDERAR APENAS REGISTROS COM status == 1 (ATIVOS).
  7) ADICIONAR UMA COLUNA “fonte_prefixo” COM O NOME DO ARQUIVO (ex.: FIXO/MOVEL/SME/OUTRO).
- UNIFICAR OS 4 ARQUIVOS EM UMA ÚNICA TABELA DE PREFIXOS.

ARQUIVO 3 – PORTABILIDADE
- DEVE INDICAR A OPERADORA ATUAL DO NÚMERO.
- PERMITA CONFIGURAR OS NOMES DAS COLUNAS, EX.:
  COLUNA_NUMERO_ATUAL = "numero"
  COLUNA_OPERADORA_ATUAL = "operadora_atual"
- PODE SER .CSV (;) OU .XLSX. TRATAR COMO STRING.

============================================================
NORMALIZAÇÃO E LÓGICA DE CASAMENTO (NÚMEROS BRASILEIROS)
============================================================

1) NORMALIZAR O NÚMERO (A_PARTIR_DE A_BUSCA E B_BUSCA):
   - MANTER APENAS DÍGITOS [0-9].
   - SE COMEÇAR COM "55", REMOVER O "55".
   - GARANTIR QUE TENHA PELO MENOS 10 DÍGITOS (DDI REMOVIDO → DDD + NÚMERO LOCAL).
   - EXTRAIR:
     DDD = 2 PRIMEIROS DÍGITOS.
     NUMERO_LOCAL = RESTANTE (8 OU 9 DÍGITOS, DEPENDENDO DO CASO).

2) CANDIDATOS DE PREFIXO (4 DÍGITOS) E ESTAÇÃO (4 DÍGITOS FINAIS):
   - ESTAÇÃO = ÚLTIMOS 4 DÍGITOS DE NUMERO_LOCAL → CONVERTER PARA INTEIRO.
   - CANDIDATO A (SEM “9” INICIAL): prefixo_cand = NUMERO_LOCAL[0:4]
   - CANDIDATO B (CON “9” INICIAL PARA MÓVEL): se NUMERO_LOCAL COMEÇA COM “9”, prefixo_cand = NUMERO_LOCAL[1:5]
   - TESTAR AMBOS OS CANDIDATOS DE PREFIXO PARA CASAR COM A TABELA:
     (codigo_nacional == DDD) E (prefixo == prefixo_cand) E (faixa_inicial <= estação <= faixa_final).

3) CRITÉRIOS EM CASO DE MÚLTIPLAS CORRESPONDÊNCIAS:
   - PRIORIZAR O MATCH COM INTERVALO MAIS ESPECÍFICO: (faixa_final - faixa_inicial) MENOR.
   - SE AINDA EMPATAR, ESCOLHER O PRIMEIRO POR ORDEM DE ARQUIVO (FIXO, DEPOIS MÓVEL, SME, OUTRO).
   - REGISTRAR EM LOG OS CASOS AMBÍGUOS.

4) OPERADORA PELO PREFIXO:
   - RETORNAR “nome_prestadora” DA LINHA CORRESPONDENTE.

5) VERIFICAÇÃO DE PORTABILIDADE:
   - BUSCAR O NÚMERO NORMALIZADO NA BASE DE PORTABILIDADE.
   - OBTER “operadora_atual”.
   - COMPARAR:
       SE operadora_atual DIFERE DE operadora_pelo_prefixo → STATUS = "PORTADO"
       CASO CONTRÁRIO → STATUS = "NÃO PORTADO"
   - SE NÚMERO NÃO EXISTIR NA BASE DE PORTABILIDADE → STATUS = "NÃO ENCONTRADO NA PORTABILIDADE"

============================================================
FUNCIONALIDADES E SAÍDA OBRIGATÓRIAS
============================================================

1) GERAR ARQUIVO FINAL: RESULTADO.xlsx
   COLUNAS MÍNIMAS:
   - numero_normalizado
   - ddd
   - prefixo_casado
   - faixa_utilizada (faixa_inicial-faixa_final)
   - fonte_prefixo (FIXO/MOVEL/SME/OUTRO)
   - operadora_pelo_prefixo
   - operadora_atual
   - status_portabilidade (PORTADO/NÃO PORTADO/NÃO ENCONTRADO NA PORTABILIDADE)
   - origem_numero (A_BUSCA OU B_BUSCA)

2) ESTRUTURA DO CÓDIGO (FUNÇÕES):
   - carregar_arquivos()
   - normalizar_numero()
   - extrair_numeros()                         # LÊ A_BUSCA/B_BUSCA EM CHUNKS
   - unificar_bases_prefixos()                 # LÊ 4 CSV, NORMALIZA E CONCATENA
   - buscar_operadora_prefixo()                # MATCH POR (DDD, PREFIXO, FAIXA)
   - verificar_portabilidade()                 # COMPARA OPERADORA DO PREFIXO X ATUAL
   - gerar_resultado()                         # ESCREVE EM CSV TEMP E CONVERTE P/ XLSX
   - main()

3) TRATAMENTO DE ERROS E LOGS:
   - ARQUIVOS INEXISTENTES OU IMPERMISSÕES (try/except COM MENSAGENS CLARAS).
   - COLUNAS AUSENTES (VALIDAÇÃO EXPLÍCITA COM ERRO AMIGÁVEL).
   - PREFIXO NÃO ENCONTRADO (STATUS: "PREFIXO NÃO ENCONTRADO").
   - NÚMERO NÃO ENCONTRADO NA PORTABILIDADE (STATUS ESPECÍFICO).
   - LOGAR VOLUMES PROCESSADOS, TEMPOS POR ETAPA E QTD DE EXCEÇÕES.

============================================================
PERFORMANCE (UM ARQUIVO É EXTREMAMENTE GRANDE)
============================================================

- LER O ARQUIVO 1 EM CHUNKS (ex.: chunksize=200_000, CONFIGURÁVEL).
- MANTER AS 4 BASES DE PREFIXO TOTALMENTE EM MEMÓRIA (SÃO RELATIVAMENTE MENORES).
- PRÉ-PROCESSAR A TABELA DE PREFIXOS:
  * CRIAR CHAVES E ÍNDICES POR (codigo_nacional, prefixo).
  * CONVERTER faixa_inicial/faixa_final PARA int.
- PARA CADA CHUNK DE NÚMEROS:
  * NORMALIZAR E DERIVAR ddd, prefixo_cand, estação.
  * FAZER MERGE PELO PAR (ddd, prefixo_cand) E FILTRAR PELA ESTAÇÃO ENTRE AS FAIXAS.
  * RESOLVER DUPLICIDADES PELOS CRITÉRIOS DEFINIDOS.
  * FAZER LOOKUP DA PORTABILIDADE POR CHAVE DO NÚMERO NORMALIZADO (USAR ÍNDICE).
  * ESCREVER RESULTADOS PARCIAIS EM CSV TEMPORÁRIO (APPEND).
- AO FINAL, CONVERTER O CSV TEMPORÁRIO PARA RESULTADO.xlsx.
- USAR TIPOS CATEGÓRICOS QUANDO POSSÍVEL E LIBERAR MEMÓRIA ENTRE ETAPAS.

============================================================
ENTREGA FINAL
============================================================

- ENTREGAR UM SCRIPT PYTHON COMPLETO, MODULARIZADO E COMENTADO.
- SUPORTAR .CSV (DELIMITADOR ‘;’) E .XLSX.
- ROBUSTO PARA GRANDES VOLUMES E COM LOGS DETALHADOS.
- RESULTADO FINAL EM RESULTADO.xlsx CONFORME ESPECIFICADO.
